## 2. AI-Generated Code Markers

### Git commit patterns

```bash
# Co-author trailers mentioning AI tools
git -C <repo> log --format="%b" | grep -i "co-authored-by.*\(copilot\|claude\|gpt\|openai\|anthropic\|cursor\|ai\)" | head -20

# Commit messages referencing AI generation
git -C <repo> log --oneline | grep -iE "(generated by|ai-assisted|copilot|cursor|claude|chatgpt|ai-generated)" | head -20

# Count AI co-author commits vs total
git -C <repo> log --format="%b" | grep -ci "co-authored-by.*\(copilot\|claude\|gpt\|openai\|anthropic\|cursor\|ai\)"
```

### Source code comments

Search for these patterns in source files (exclude `node_modules`, `.git`, `vendor`, `dist`):

```bash
grep -rn --include="*.{ts,js,py,rb,go,rs,java,jsx,tsx}" \
  -iE "(generated by|auto-generated|ai-generated|created by copilot|created by claude|created by gpt)" \
  <repo> | head -30
```

## 4. AI-Related Environment Variables

Search for references in source code, configs, `.env.example`, and documentation:

```bash
grep -rn --include="*.{ts,js,py,rb,go,rs,java,env,yml,yaml,toml,json,md}" \
  -E "(OPENAI_API_KEY|ANTHROPIC_API_KEY|CLAUDE_API_KEY|HUGGINGFACE_TOKEN|HF_TOKEN|COHERE_API_KEY|REPLICATE_API_TOKEN|GOOGLE_AI_API_KEY|MISTRAL_API_KEY|PINECONE_API_KEY|QDRANT_API_KEY|TOGETHER_API_KEY|GROQ_API_KEY|AZURE_OPENAI_API_KEY|AZURE_OPENAI_ENDPOINT|LANGCHAIN_API_KEY|LANGSMITH_API_KEY)" \
```

## 8. Scoring Dimensions

### AI Tool Adoption Level
| Level | Criteria |
|-------|----------|
| **None** | No AI tool configs, no LLM dependencies, no AI commit markers |
| **Exploratory** | 1-2 AI tool configs present OR minor LLM dependency (dev-only) |
| **Integrated** | Active AI tool configs + LLM dependencies in production code + regular AI commit markers |
| **AI-Native** | Multiple AI tool configs + LLM dependencies are core to business logic + majority of recent commits have AI markers |

### AI Dependency Risk
| Level | Criteria |
|-------|----------|
| **Low** | No LLM API dependencies, or LLM used only in dev tooling |
| **Medium** | LLM API calls present but not in critical user-facing paths; fallbacks exist |
| **High** | LLM API calls in critical paths (auth, payments, core logic); no fallback handling; single-provider lock-in |

### Code Provenance Clarity
| Level | Criteria |
|-------|----------|
| **Clear** | AI contributions marked consistently (co-author trailers); governance docs present; easy to distinguish AI vs human code |
| **Partial** | Some AI markers present but inconsistent; no formal policy |
| **Opaque** | Evidence of AI tool usage (configs present) but no commit markers or governance; impossible to determine what was AI-generated |

### AI Governance Maturity
| Level | Criteria |
|-------|----------|
| **None** | No AI-related docs, no PR template requirements, no policy |
| **Informal** | AI tools in use with ad-hoc practices; no written policy |
| **Documented** | Written AI policy or CONTRIBUTING.md guidelines; PR template mentions AI |
| **Enforced** | Policy + CI checks or review requirements for AI-generated code |
